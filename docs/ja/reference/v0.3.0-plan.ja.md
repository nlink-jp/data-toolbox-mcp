# v0.3.0 開発計画

> Status: Draft
> Date: 2026-06-06
> 前提 ADR: ADR-0008 (`attach_files`) / ADR-0009 (`load_from_work`)

v0.2.0 Claude Desktop 実機検証から派生したフィードバックのうち、優先度の高い「受け渡しの最後の一歩」2 件を実装する。`feedback_adr_before_impl` 規律に従い、ADR-0008 / ADR-0009 を Accepted にしてから着手。

## 1. Goals (v0.3.0 完成条件)

- 8 MCP ツール: 既存 6 + 新規 `attach_files` + `load_from_work`
- `attach_files` が PNG/JPG/SVG をインライン画像として、CSV/JSON/MD を text として返す。サイズ上限超過は metadata-only に降格
- `load_from_work` が `/work/<sub>` 表記で sandbox 内 CSV/JSON/Parquet を直接 DuckDB table 化
- `internal/mcpserver` に `RawResult` 型を追加し複数 content block 返却に対応 (attach_files の前提)
- ユニット + 統合 + E2E テストすべて pass、path traversal 攻撃シナリオ含む
- README × 2 / AGENTS / architecture / CHANGELOG / samples が新ツールを反映
- v0.3.0 タグ + 5 platform release + darwin notarize

## 4. Open questions

- **`attach_files` の text content overflow 表現**: head + tail で挟むか、head のみ + 省略マーカーか → head + tail (より文脈保持) を MVP に
- **metadata-only の sha256 計算コスト**: 100 MiB ファイル等で数秒かかる可能性 → 上限 100 MiB を超えるファイルは sha256 を省略 (size + path のみ)、文言に "[sha256 omitted: file too large]" を入れる
- **`load_from_work` で `/work/subdir/` のような階層パスを許すか**: 許す (`filepath.Clean` + prefix check で安全)。LLM が自然に書ける
