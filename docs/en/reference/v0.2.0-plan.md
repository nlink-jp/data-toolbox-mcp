## 2. Work breakdown (5 tracks)

### Track 1: Dockerfile + matplotlibrc extension (implements ADR-0007)

- `runtime/Dockerfile`:
  ```dockerfile
  FROM python:3.12-slim

  RUN apt-get update && apt-get install -y --no-install-recommends \
          fonts-noto-cjk ca-certificates \
      && rm -rf /var/lib/apt/lists/*

  RUN pip install --no-cache-dir \
          duckdb~=1.1 pandas~=2.2 polars~=1.8 pyarrow~=18.0 \
          matplotlib~=3.10 Pillow~=11.0

  RUN mkdir -p /etc/matplotlib && \
      printf 'font.family: sans-serif\nfont.sans-serif: Noto Sans CJK JP, DejaVu Sans, Arial, Liberation Sans\naxes.unicode_minus: False\n' \
        > /etc/matplotlib/matplotlibrc
  ENV MATPLOTLIBRC=/etc/matplotlib/matplotlibrc

  RUN useradd -m -u 1000 toolbox
  USER 1000:1000
  WORKDIR /work
  CMD ["sleep", "infinity"]
  ```

## 4. Open questions

- **Strictness of the manifest-drift test**: is package-name set equality enough, or should we also assert that the installed version's prefix matches the pin (e.g. installed `duckdb` starts with `1.1.`)? MVP says name-set only; tighten in Phase 3 alongside TTL GC.
- **`list_workspaces` empty disk**: behavior when workspace_dir itself doesn't exist? Return `[]` (not an error), gated by `os.IsNotExist` check.
- **`describe_runtime`'s `notes` field**: how to i18n? Defer to Phase 3; English-only is fine for v0.2.0.
